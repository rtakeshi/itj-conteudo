
# Introdução ao Mercado de Dados

## Objetivo do bloco

Neste bloco vamos mostrar, de forma simples e prática, como é o trabalho com Banco de Dados, Engenharia de Dados e Ciência de Dados no mercado. Tem exemplos de tarefas, papéis e problemas do dia a dia — sem termos difíceis.

## O que você vai ver

- Relatos de atividades típicas em Banco de Dados, Engenharia de Dados e Ciência de Dados.
- Como equipes de dados podem ser organizadas e o que isso muda no dia a dia.
- Aplicações práticas do conteúdo em Engenharia de Software, times de Data e na rotina de DBAs.

## Exemplos de composição de equipes de dados

As equipes de dados podem ser formadas de maneiras diferentes, dependendo do tamanho da empresa e do tipo de problema que ela resolve.

Abaixo estão alguns exemplos comuns de composição de equipes no mercado.

### Equipe de Dados Básica
Indicada para empresas pequenas ou projetos iniciais.

- Engenheiro de Dados  
- Analista de Dados  

Essa equipe cuida da coleta, organização e análise básica dos dados.

---

### Equipe de Engenharia de Dados
Focada em infraestrutura e pipelines de dados.

- Vários Engenheiros de Dados  
- Profissional de DataOps  

Essa equipe constrói e mantém pipelines, bancos de dados e garante que os dados cheguem corretamente aos sistemas e análises.

---

### Equipe de Dados com Produto
Comum em empresas orientadas a produto digital.

- Engenheiros de Dados  
- Analista de Dados  
- Product Manager (PM)  

O Product Manager ajuda a definir prioridades, enquanto o time de dados trabalha para atender necessidades do produto.

---

### Equipe de Business Intelligence (BI)
Focada em relatórios e apoio à tomada de decisão.

- Analista de BI  
- Engenheiro de Dados  
- Stakeholders de negócio  

Essa equipe transforma dados em dashboards e relatórios para gestores e líderes.

---

### Equipe de Ciência de Dados
Voltada para análises avançadas e modelos preditivos.

- Cientista de Dados  
- Engenheiro de Dados  
- Analista de Dados  

O engenheiro prepara os dados, o cientista analisa e cria modelos, e o analista valida e apresenta resultados.

---

### Equipe de Plataforma de Dados
Responsável por criar ferramentas internas para outros times.

- Engenheiros de Dados  
- Profissional de DataOps  
- Engenheiro de Software  

Essa equipe cria bases, pipelines e ferramentas para que outros times usem dados com mais facilidade.

---

### Equipe Híbrida (Central + Produto)
Modelo comum em empresas médias e grandes.

- Time central de Dados (Engenheiros de Dados, DataOps)  
- Profissionais de dados alocados em times de produto  

O time central define padrões e ferramentas, enquanto os times de produto usam dados no dia a dia.

---

### Equipe Completa de Dados (Empresa Grande)
Estrutura encontrada em grandes empresas de tecnologia.

- Engenheiros de Dados  
- Cientistas de Dados  
- Analistas de Dados  
- Profissionais de DataOps  
- Product Manager  
- DBA  

Cada profissional tem um papel específico para garantir qualidade, análise e uso eficiente dos dados.


## Aplicações do conteúdo

- Engenharia de Software: usar dados nos sistemas, integrar pipelines e manter o código que fala com o banco.
- Engenharia de Dados: construir pipelines que movem e limpam dados e garantir que os dados cheguem certos ao lugar.
- Ciência de Dados: preparar dados para análises, testar ideias e colocar modelos para rodar.
- DBA: cuidar do banco (tabelas e índices), melhorar desempenho, fazer backups e cuidar da segurança básica.


## Glossário Técnico 
Este glossário apresenta termos comuns do mercado de dados explicados de forma simples, com foco em alunos do ensino médio técnico.

### Modelagem Relacional
Forma tradicional de organizar dados em tabelas.
Os dados são separados por assunto e ligados por chaves, evitando repetição de informações.
Muito usada em sistemas comerciais, escolares e administrativos.

### Modelagem para BI (Dimensional)
Forma de organizar dados pensando em análise e relatórios.
Divide os dados em fatos (números) e dimensões (contexto).
Usada em dashboards e relatórios gerenciais.

### Pipeline de Dados
Conjunto de etapas que os dados percorrem desde a origem até o uso final.
Envolve coleta, organização e armazenamento dos dados.
Garante que os dados cheguem corretos ao destino.

### Modelagem Estatística
Uso de conceitos básicos de estatística para entender dados.
Ajuda a identificar padrões, médias e comportamentos.
Muito usada em análises e ciência de dados.

### Gestão de Banco de Dados
Atividades para manter o banco funcionando corretamente.
Inclui criação de tabelas, segurança, backup e desempenho.

### Banco de Dados Relacional
Banco de dados organizado em tabelas relacionadas entre si.
Evita duplicação de informações.
Usado em sistemas tradicionais.

### Banco de Dados Não Relacional
Banco de dados que não usa tabelas tradicionais.
Os dados podem ter formatos variados.
Usado em sistemas com grande volume de dados.

### Tabela
Estrutura que armazena dados em linhas e colunas.

### Registro
Conjunto completo de informações dentro de uma tabela.
Cada linha representa um registro.

### Campo (Coluna)
Tipo de informação armazenada em uma tabela.
Exemplo: nome, idade, nota.

### Chave Primária
Campo que identifica um registro de forma única.
Evita registros duplicados.

### Chave Estrangeira
Campo que liga uma tabela a outra.
Permite relacionar informações.

### Consulta (Query)
Pedido feito ao banco de dados para buscar informações.
Normalmente escrita em SQL.

### SQL
Linguagem usada para criar, consultar e alterar dados em bancos relacionais.

### ETL
Processo de dados dividido em três etapas:
Extract (extrair), Transform (organizar) e Load (carregar).

### Data Warehouse
Banco de dados voltado para análise.
Armazena dados históricos organizados.

### Data Lake
Local onde os dados são armazenados em formato bruto.
Usado quando o uso dos dados ainda não está definido.

### Dashboard
Painel visual com gráficos e indicadores.
Facilita a visualização de informações importantes.

### Qualidade de Dados
Práticas para garantir que os dados estejam corretos e completos.
Dados ruins geram decisões erradas.

### Governança de Dados
Regras e processos para uso correto dos dados.
Garante organização, segurança e responsabilidade.

### Big Data
Grande volume de dados gerados rapidamente.
Exige ferramentas especiais para processamento.

### Ciência de Dados
Área que usa dados, estatística e programação para gerar conhecimento.

### Engenharia de Dados
Área responsável por construir pipelines e estruturas de dados.

### DBA
Profissional que cuida do banco de dados.
Responsável por segurança, desempenho e backup.


## Conclusão

Ao longo deste bloco, vimos como o mercado de dados funciona na prática, quais são os principais papéis envolvidos e como as equipes de dados podem ser organizadas dentro das empresas.

Engenharia de Dados, Ciência de Dados, BI e Banco de Dados não são áreas isoladas. Elas trabalham juntas para transformar dados brutos em informação útil para sistemas, produtos e tomada de decisão.

No entanto, é importante entender que nada disso existe sem uma base sólida. Pipelines, análises, dashboards e modelos só funcionam bem quando os dados estão corretamente armazenados, organizados e relacionados.

Por isso, antes de avançar para temas mais modernos do mercado de dados, precisamos voltar ao começo e garantir que os fundamentos de Banco de Dados estejam claros.

Conceitos como tabelas, registros, chaves primárias e estrangeiras, modelagem relacional e consultas em SQL são a base de tudo. Esses conhecimentos, vistos em Banco de Dados III e IV, continuam sendo utilizados diariamente por engenheiros de dados, analistas, cientistas e DBAs.

Dominar os fundamentos de Banco de Dados não é algo ultrapassado. Pelo contrário: é o que permite evoluir com segurança para áreas como Engenharia de Dados, BI e Ciência de Dados ao longo da carreira.

A partir daqui, o foco do curso será reforçar esses fundamentos, conectando teoria e prática, para que os alunos entendam não apenas como criar um banco de dados, mas por que ele é essencial no funcionamento dos sistemas e no mercado de tecnologia.
