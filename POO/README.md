# Programação Orientada a Objetos I e II

**Curso:** Técnico em Informática  
**Disciplina:** Programação Orientada Objeto I e II  
**Período Letivo:** 3º Série  
**Total de Aulas / Total de Horas:** 120 / 100H

## Ementa
Introdução à orientação a objetos. Conceitos básicos e terminologia de Programação orientada a objetos. Linguagens típicas orientadas a objetos. Desenvolvimento de aplicações em uma linguagem orientada a objetos com acesso à Banco de Dados.

## Competências / Habilidades
- Projetar sistemas de informação, selecionando linguagens de programação e ambientes de desenvolvimento de acordo com as especificidades do projeto.
- Codificar programas orientados a objetos.
- Utilizar ambientes de desenvolvimento para desenvolver sistemas orientado a objeto.
- Conectar aplicações com banco de dados.
- Aplicar técnicas de orientação a objetos.

## Bases Científico-Tecnológicas
1. Fundamentos
   - (a) Paradigmas de programação: sequencial e estruturada
   - (b) História e princípios da linguagem Java
   - (c) Kit de Desenvolvimento Java (JDK) e Ambiente integrado de desenvolvimento (IDE)
   - (d) Tipos primitivos, estruturas de decisão e repetição, vetores uni e multidimensionais
   - (e) Leitura de dados do dispositivo de entrada padrão com classe Scanner
   - (f) Principais métodos da classe String
   - (g) Argumentos de linha de comando

2. Sistema de controle de versão
   - (a) Motivação para uso de sistemas de controle de versão
   - (b) Sistema de controle de versão centralizado (SVN) vs sistema de controle de versão distribuído (git)
   - (c) Apresentação da ferramenta git
     - i. Como iniciar um repositório git e registrar as modificações (commit)
     - ii. Introdução a linguagem de marcação Markdown
     - iii. Comandos git: status, add, commit, reset, diff, log, rm e mv
   - (d) Trabalhando com repositórios remotos no Github
     - i. Evitando o controle de alguns arquivos por meio do .gitignore
     - ii. Clonando repositório remoto e sincronizando repositório local com o remoto
     - iii. Trabalhando de forma colaborativa
     - iv. Comandos git: remote, clone, push, pull e fetch
   - (e) Trabalhando com ramos
     - i. Fluxos de trabalho com ramos em projetos colaborativos
     - ii. Resolvendo conflitos em uma mesclagem (merge conflict)
     - iii. Comandos git: branch, checkout, merge e rebase

3. Introdução ao paradigma da orientação a objetos
   - (a) Processos de abstração e representação
   - (b) Classes, objetos e membros de classe (atributos e métodos)
   - (c) Encapsulamento de dados
   - (d) Modelagem de classes
   - (e) Métodos construtores
   - (f) Sobrecarga de métodos
   - (g) Membros estáticos e constantes
   - (h) Modificadores de acesso e palavras reservadas

4. APIs Java
   - (a) Tratamento de exceções
   - (b) Coleções (lista, tabela de dispersão, conjuntos, pilha e fila)

5. Associação entre classes
   - (a) Diagrama de classes UML
   - (b) Dependência, agregação e composição

6. Testes de unidade e documentação de código
   - (a) Desenvolvimento guiado a testes
   - (b) Documentação com JavaDOC

7. Herança
   - (a) Sobrescrita de métodos
   - (b) Diagrama de classes UML

8. Classe abstrata, interface e polimorfismo
   - (a) Conceito sobre herança múltipla
   - (b) Implementação de interfaces
   - (c) Diagrama de classes UML

9. Uso de APIs Java
   - (a) Trabalhando com arquivos texto e binários
   - (b) Enum e tipos genéricos
   - (c) Programação concorrente com threads

10. Processo de desenvolvimento de software
   - (a) Conceitos sobre processo unificado
   - (b) Diagrama de casos de uso
   - (c) Diagrama de sequência
   - (d) Diagrama de colaboração
   - (e) Diagrama de atividades

11. Linguagem de programação Python
   - (a) Ferramentas e IDE
   - (b) Estruturas básicas da linguagem
   - (c) Orientação a objetos
